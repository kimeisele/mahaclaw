# CLAUDE.md — Maha Claw Project Memory

## What this project IS

Maha Claw is a **pure Python bridge** that translates flat OpenClaw intents into
the RAMA/NADI federation transport used by the kimeisele agent ecosystem.

It is NOT a template, NOT a concept doc, NOT a chat project.
It is a working daemon that reads JSON, enriches it, and writes wire-compatible
DeliveryEnvelopes to `nadi_outbox.json`.

## Current state

**PoC is DONE and all 26 tests pass.** The 5-gate pipeline works end-to-end:

| Gate     | File                  | Status  | What it does                          |
|----------|-----------------------|---------|---------------------------------------|
| PARSE    | `mahaclaw/intercept.py` | Done  | Validate incoming JSON intent         |
| VALIDATE | `mahaclaw/tattva.py`    | Done  | 5D Tattva affinity → element/zone     |
| EXECUTE  | `mahaclaw/rama.py`      | Done  | 7-layer RAMA signal encoding          |
| RESULT   | `mahaclaw/lotus.py`     | Done  | O(1) route resolution + buddy_bubble  |
| SYNC     | `mahaclaw/envelope.py`  | Done  | DeliveryEnvelope → nadi_outbox.json   |

Daemon: `mahaclaw/daemon.py` — asyncio Unix socket server.
Tests: `tests/test_mahaclaw.py` — 26 tests covering every gate + e2e + socket.

## Hard rules

1. **Pure Python stdlib only.** No pip dependencies at runtime. Not requests, not aiohttp, not pydantic. If you need HTTP, use `curl` subprocess like the rest of the federation.
2. **Short commit messages.** 3-8 words max. No essays in git log. Example: `"add tattva classifier"`, `"fix lotus fallback"`.
3. **No new files without a reason.** Edit existing files first. Don't create utils/helpers/abstractions for one-off things.
4. **Tests must pass before push.** Run `python -m pytest tests/test_mahaclaw.py -q` and confirm 0 failures.
5. **Wire compatibility is sacred.** The envelope format in `envelope.py` matches `steward-federation/nadi/*.json` and `agent-internet/transport.py:DeliveryEnvelope`. Do not change field names, do not drop fields, do not invent new envelope schemas.
6. **Do not touch `.well-known/` files directly.** They are auto-generated by workflows.

## Architecture decisions (do not revisit)

- Unix domain socket for local IPC, not HTTP. Faster, simpler, no port conflicts.
- Static `AFFINITY_MAP` in tattva.py for PoC. Known limitation, see roadmap.
- NADI type derived from dominant Tattva element, not hardcoded per intent.
- MahaHeader hex = SHA-256 of `source:target:operation:nadi_type:priority:ttl_ms`, truncated to 32 chars. Matches upstream `steward_protocol_compat.py`.
- Parampara vector = `(position + 1) * 37`. Always divisible by 37 = connected.
- Default fallback for unknown intents: position 9 (Prahlada / EXEC_SERVICE), element vayu (general).

## Upstream wire format reference

Envelopes written to `nadi_outbox.json` must contain these fields:

```
source, source_city_id, target, target_city_id, operation, payload,
envelope_id, correlation_id, id, timestamp, priority, ttl_s, ttl_ms,
nadi_type, nadi_op, nadi_priority, maha_header_hex
```

NADI types: `prana`, `apana`, `vyana`, `udana`, `samana`
NADI ops: `receive`, `send`, `cache`, `process`, `validate`, `request`, `delegate`, `connect`, `commit`
Guna priorities: `tamas` (1), `rajas` (5), `sattva` (8), `suddha` (10)

## Roadmap — what comes next

### Phase 2: Dynamic Tattva classification
The static `_AFFINITY_RULES` in `tattva.py` is the main bottleneck. When OpenClaw
generates novel intents that don't match any keyword, they all fall back to vayu.
Replace with a scoring function that can be extended — either rule-based config
file or (later) Jiva model integration from steward's Sankhya-25 engine.

### Phase 3: Reverse path (inbox → OpenClaw response)
Currently one-directional: OpenClaw → NADI outbox. Need to watch `nadi_inbox.json`
(or a dedicated mahaclaw inbox) and translate federation responses back into
OpenClaw response format.

### Phase 4: Daemon hardening
- Systemd unit file or `__main__.py` entry point
- Structured JSON logging
- Health check endpoint (could be a second socket or a simple file)
- Outbox rotation / size limits
- Connection rate limiting

### Phase 5: Federation registration
- Update `docs/authority/capabilities.json` to declare `mahaclaw_bridge` skill
- Update `docs/authority/charter.md` with Maha Claw role
- Register in agent-city as Engineering zone citizen
- Publish capability: `produces: ["nadi_envelope"]`, `consumes: ["openclaw_intent"]`

## How to run

```bash
# Tests
python -m pytest tests/test_mahaclaw.py -q

# Start daemon
python -m mahaclaw.daemon

# Send a test intent (separate terminal)
echo '{"intent":"inquiry","target":"agent-research","payload":{"q":"test"}}' | \
  python -c "import socket,sys; s=socket.socket(socket.AF_UNIX,socket.SOCK_STREAM); s.connect('mahaclaw.sock'); s.sendall(sys.stdin.buffer.read()); s.shutdown(1); print(s.recv(65536).decode())"

# Check outbox
python -c "import json; print(json.dumps(json.load(open('nadi_outbox.json')),indent=2))"

# Inspect routing table
python -c "from mahaclaw.lotus import buddy_bubble; import json; print(json.dumps(buddy_bubble(),indent=2))"
```
